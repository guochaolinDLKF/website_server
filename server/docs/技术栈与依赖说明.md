## 二、核心依赖详解

### 1. MyBatis Plus — 数据库操作

**作用**：让你不写 SQL 就能操作数据库。

**基本用法**：

```java
// 定义一个实体类（对应数据库表）
@Data
@TableName("user")           // 指定表名
public class User {
    @TableId(type = IdType.AUTO)  // 主键自增
    private Long id;
    private String phoneCode;
    private String nickName;
}

// 定义一个 Mapper
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 继承 BaseMapper 后，自动拥有这些方法：
    // insert(), deleteById(), updateById(), selectById(), selectList() ...
}

// 在 Service 中使用
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> {
    public User findUser(Long id) {
        return getById(id);  // 等价于 SELECT * FROM user WHERE id = ?
    }
}
```

**条件查询**（复杂查询时不写 SQL）：

```java
// 查询手机号为 138xxx 的用户
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(User::getPhoneCode, "13800138000");
User user = getOne(wrapper);

// 等价于：SELECT * FROM user WHERE phone_code = '13800138000'
```

**分页查询**（需在 MybatisPlusConfig 中注册分页插件）：

```java
Page<User> page = new Page<>(1, 10);  // 第1页，每页10条
Page<User> result = userMapper.selectPage(page, null);
```

### 2. Redis + Redisson — 缓存与分布式锁

#### 基本缓存操作

```java
@Autowired
private StringRedisTemplate stringRedisTemplate;

// 存入
stringRedisTemplate.opsForValue().set("key", "value", Duration.ofMinutes(5));

// 取出
String value = stringRedisTemplate.opsForValue().get("key");

// 删除
stringRedisTemplate.delete("key");
```

#### 分布式锁（@RedissonLock 注解）

**作用**：防止多个请求同时操作同一资源（如防止用户重复下单）。

```java
// 在方法上加 @RedissonLock
@RedissonLock(
    key = "order:create:#{#userId}",   // 锁的 key，支持 SpEL 表达式
    waitTime = 30,                      // 最多等 30 秒
    leaseTime = 60,                     // 锁自动释放时间 60 秒
    message = "请勿重复操作"
)
public void createOrder(Long userId) {
    // 这个方法同时只能有一个线程在执行（同一 userId）
}
```

### 3. JetCache — 二级缓存

**什么是二级缓存**：数据先查本地内存（Caffeine），没有再查 Redis，再没有再查数据库。比单用 Redis 更快。

**用法**：

```java
// 查询时自动缓存（方法返回值会被缓存）
@Cached(name = "userCache", key = "#userId", expire = 3600)
public User getUserById(Long userId) {
    return getById(userId);  // 第一次查数据库，之后从缓存取
}

// 更新时自动刷新缓存
@CacheUpdate(name = "userCache", key = "#user.id", value = "#user")
public void updateUser(User user) {
    updateById(user);
}

// 删除时自动清除缓存
@CacheInvalidate(name = "userCache", key = "#userId")
public void deleteUser(Long userId) {
    removeById(userId);
}
```

**缓存策略**：
- **本地缓存**：Caffeine，存在 JVM 内存中，极快
- **远程缓存**：Redis，所有服务实例共享
- **刷新机制**：`@CacheRefresh(refresh = 2400)` — 每 40 分钟自动刷新（不等过期才刷新）

### 4. Sa-Token — 权限认证

**核心概念**：
- 用户登录后，Sa-Token 生成一个 Token（一串随机字符串）
- Token 存到 Redis 中，默认 3 天过期
- 每次请求携带 Token，服务端校验是否有效

**登录**：

```java
// 执行登录，Sa-Token 自动生成 Token
StpUtil.login(userId);

// 获取 Token 值
String token = StpUtil.getTokenValue();
```

**鉴权**：

```java
// 检查是否登录（未登录会抛异常，由全局异常处理器转成 401）
StpUtil.checkLogin();

// 获取当前登录用户 ID
Long userId = StpUtil.getLoginIdAsLong();

// 登出
StpUtil.logout();
```

**拦截器配置**：项目已配置好，白名单接口在 [WebConfig.java](file:///D:/work_space/website_server/server/src/main/java/com/ydzz/config/WebConfig.java) 中：

```java
.notMatch(
    "/api/user/login",        // 登录接口，不需 Token
    "/swagger-ui/**",         // Swagger 文档，不需 Token
    "/v3/api-docs/**",
    "/error",
    "/favicon.ico"
)
```

**新增公开接口**：直接在 `.notMatch(...)` 中添加路径即可。

### 5. Token 加密机制

本项目的 Token 经过了 **AES 加密**，不是明文传输：

```
原始 Token（Sa-Token 生成） → AES 加密 → 返回给客户端
客户端请求时携带密文      → AES 解密 → 还原原始 Token → 验证
```

加密密钥在 [CustomTokenParser.java](file:///D:/work_space/website_server/server/src/main/java/com/ydzz/config/CustomTokenParser.java) 中，生产环境请务必更换密钥。

### 8. SnowflakeIdGenerator — 分布式 ID

**作用**：生成全局唯一的 ID，基于 Twitter Snowflake 算法，适合分布式环境。

```java
@Autowired
private SnowflakeIdGenerator idGenerator;

// 生成订单号
String orderNo = idGenerator.generateOrderNo();

// 生成支付单号
String paymentNo = idGenerator.generatePaymentNo();
```

特性：
- 生成的 ID 是递增的（趋势递增，不是绝对递增）
- 即使多台服务器同时生成，也不会重复
- 长度 19 位数字

## 三、可选依赖（按需启用）

### RocketMQ — 消息队列

**用途**：异步处理耗时操作（如发送通知、处理支付回调）。

**启用方式**：在配置文件中添加：

```yaml
rocketmq:
  name-server: 127.0.0.1:9876    # RocketMQ 服务地址
  producer:
    group: your-producer-group
```

### XXL-JOB — 定时任务

**用途**：定时执行批处理任务（如每天清理过期数据）。

**启用方式**：取消 [XxlJobConfig.java](file:///D:/work_space/website_server/server/src/main/java/com/ydzz/config/XxlJobConfig.java) 的注释，并在配置文件中添加：

```yaml
xxl:
  job:
    admin:
      addresses: http://127.0.0.1:8080/xxl-job-admin
    executor:
      appname: your-app
      port: 9999
```

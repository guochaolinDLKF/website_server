# 开发扩展指南

## 一、添加一个新的数据库表（完整流程）

假设你要添加一个"文章"功能。需要创建 5 个文件：

### 第 1 步：创建 Entity（实体类）

```java
// entity/Article.java
package com.ydzz.entity;

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
import java.time.LocalDateTime;

@Data
@TableName("article")        // 对应数据库表名
public class Article {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String title;           // 标题
    private String content;         // 内容
    private Long authorId;          // 作者 ID

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;
}
```

```java
// service/impl/ArticleServiceImpl.java
package com.ydzz.service.impl;

import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.ydzz.entity.Article;
import com.ydzz.mapper.ArticleMapper;
import com.ydzz.service.ArticleService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

@Slf4j
@Service
public class ArticleServiceImpl
        extends ServiceImpl<ArticleMapper, Article>
        implements ArticleService {

    @Override
    public Article createArticle(Long authorId, String title, String content) {
        Article article = new Article();
        article.setAuthorId(authorId);
        article.setTitle(title);
        article.setContent(content);
        save(article);   // MyBatis Plus 自带的方法
        return article;
    }
}
```

### 第 5 步：创建 Controller

```java
// controller/ArticleController.java
package com.ydzz.controller;

import cn.dev33.satoken.stp.StpUtil;
import com.ydzz.common.Result;
import com.ydzz.dto.ArticleCreateRequest;
import com.ydzz.dto.ArticleResponse;
import com.ydzz.entity.Article;
import com.ydzz.service.ArticleService;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/article")
@RequiredArgsConstructor
@Tag(name = "文章管理", description = "文章的增删改查")
public class ArticleController {

    private final ArticleService articleService;

    @Operation(summary = "创建文章")
    @PostMapping
    public Result<ArticleResponse> create(@Valid @RequestBody ArticleCreateRequest request) {
        Long userId = StpUtil.getLoginIdAsLong();
        Article article = articleService.createArticle(userId,
                request.getTitle(), request.getContent());

        ArticleResponse response = new ArticleResponse();
        response.setId(article.getId());
        response.setTitle(article.getTitle());
        response.setAuthorId(article.getAuthorId());
        response.setCreateTime(article.getCreateTime());
        return Result.success("创建成功", response);
    }
}
```

> **注意**：`/api/article/*` 的接口默认需要登录才能访问（被拦截器保护），不需要额外配置。

### 建表 SQL

```sql
CREATE TABLE article (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    title       VARCHAR(200) NOT NULL,
    content     TEXT,
    author_id   BIGINT NOT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 总结：添加新功能的文件清单

```
新增功能需要创建的文件：
├── entity/Xxx.java          ← 实体类
├── mapper/XxxMapper.java    ← 数据访问（可选，可用 BaseMapper）
├── dto/XxxRequest.java      ← 请求参数
├── dto/XxxResponse.java     ← 返回数据
├── service/XxxService.java  ← 服务接口
├── service/impl/XxxServiceImpl.java ← 服务实现
└── controller/XxxController.java    ← 控制器

可选：
├── mapper/XxxMapper.xml     ← 复杂 SQL 时才需要
├── enums/XxxStatus.java     ← 状态枚举
```

## 三、使用缓存加速查询

对于**读多写少**的数据（如用户信息），加缓存可以大幅提升性能：

```java
// 查询时自动缓存（1 小时过期，40 分钟自动刷新）
@Cached(name = "articleCache", key = "#id", expire = 3600)
@CacheRefresh(refresh = 2400)
public Article getArticleById(Long id) {
    return getById(id);
}

// 更新时自动刷新缓存
@CacheUpdate(name = "articleCache", key = "#article.id", value = "#article")
public void updateArticle(Article article) {
    updateById(article);
}

// 删除时自动清除缓存
@CacheInvalidate(name = "articleCache", key = "#id")
public void deleteArticle(Long id) {
    removeById(id);
}
```

## 四、使用分布式锁防止并发问题

需要修改共享数据的操作，加锁避免并发冲突：

```java
@RedissonLock(
    key = "article:update:#{#articleId}",
    waitTime = 10,
    leaseTime = 30,
    message = "操作太频繁，请稍后再试"
)
@Transactional(rollbackFor = Exception.class)
public void updateArticle(Long articleId, String newTitle) {
    Article article = getById(articleId);
    article.setTitle(newTitle);
    updateById(article);
}
```

## 五、使用事务保证数据一致性

需要同时操作多张表时，确保要么全成功、要么全回滚：

```java
@Transactional(rollbackFor = Exception.class)
public void transferPoints(Long fromUserId, Long toUserId, Integer points) {
    // 扣减积分
    User fromUser = getById(fromUserId);
    fromUser.setPoints(fromUser.getPoints() - points);
    updateById(fromUser);

    // 增加积分
    User toUser = getById(toUserId);
    toUser.setPoints(toUser.getPoints() + points);
    updateById(toUser);

    // 记录流水
    PointsLog log = new PointsLog();
    log.setFromUserId(fromUserId);
    log.setToUserId(toUserId);
    log.setPoints(points);
    pointsLogMapper.insert(log);

    // 以上 4 个操作，任何一个失败，全部回滚
}
```

## 六、统一错误处理

**不要**在每个方法里写 try-catch，直接抛异常：

```java
// ❌ 不好的写法
public User getUser(Long id) {
    try {
        User user = getById(id);
        if (user == null) {
            return null;  // 调用方还得判断 null
        }
        return user;
    } catch (Exception e) {
        log.error("出错", e);
        return null;
    }
}

// ✅ 好的写法
public User getUser(Long id) {
    User user = getById(id);
    BusinessAssert.notNull(user, ErrorCode.UserIsNull);
    // 后续直接用 user，不需要判空
    return user;
}
```

`BusinessAssert.notNull()` 会在 user 为空时抛出 `BusinessException`，由全局异常处理器 `GlobalExceptionHandler` 统一转成：

```json
{"code": -5, "message": "找不到这样的用户", "data": null}
```

## 八、添加定时任务

在任意 `@Component` 类中加 `@Scheduled` 注解：

```java
@Component
public class DailyReportJob {

    @Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨 2 点执行
    public void generateDailyReport() {
        log.info("开始生成日报...");
        // 你的业务逻辑
    }

    @Scheduled(fixedRate = 60000)  // 每 60 秒执行一次
    public void checkSomething() {
        log.info("定时检查...");
    }
}
```

## 九、如何调试

1. **SQL 日志**：`application-local-dev.yml` 中配置了 `log-impl: StdOutImpl`，所有 SQL 会打印在控制台
2. **Druid 监控**：访问 `http://localhost:8000/druid/`，查看慢 SQL
3. **Swagger 测试**：访问 `http://localhost:8000/swagger-ui.html`，在线调试接口
4. **日志级别**：在配置文件中调整

```yaml
logging:
  level:
    com.ydzz: debug          # 你自己的代码用 DEBUG
    org.springframework: info # Spring 框架用 INFO
```

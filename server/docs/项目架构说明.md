# 项目架构说明

## 一、这个项目是什么

一个基于 **Spring Boot 3** 的后端通用框架，提供了用户认证、数据库操作、缓存、分布式锁等开箱即用的能力。你可以在这个基础上快速开发任何业务功能。

### 请求处理全流程（一次登录为例）

```
客户端                         服务端
  │                              │
  │  POST /api/user/login        │
  │  {"phoneCode":"138...",      │
  │   "verifyCode":"123456"}     │
  │ ──────────────────────────>  │
  │                              │ ① 拦截器：不在白名单，但 /api/user/login 被排除，放行
  │                              │ ② UserController.login() 接收请求
  │                              │ ③ UserServiceImpl.loginByPhone()
  │                              │    ├─ getByPhone() → 查 JetCache → 未命中 → 查 MySQL
  │                              │    ├─ 用户不存在 → 创建新用户 → MyBatis Plus save()
  │                              │    ├─ StpUtil.login(userId) → Sa-Token 生成原始 Token
  │                              │    ├─ AES 加密 Token → 返回加密后的 Token
  │                              │    └─ JetCache 缓存用户信息
  │                              │ ④ 返回 {"token":"加密字符串", "userInfo":{...}}
  │  {"token":"xxx","userInfo"}  │
  │ <──────────────────────────  │
  │                              │
  │  后续请求：                    │
  │  GET /api/user/info          │
  │  Authorization: Bearer xxx   │
  │ ──────────────────────────>  │
  │                              │ ① CustomTokenInterceptor → 去 Bearer 前缀 → AES 解密
  │                              │ ② SaInterceptor → checkLogin() → 验证通过
  │                              │ ③ UserController.getUserInfo() → 执行业务逻辑
```

## 四、环境切换

**只需要改 `application.yml` 中的一行：**

```yaml
spring:
  profiles:
    active: local-dev   # 改这里：local-dev / dev / prod / arraignment
```

也可以启动时用命令行参数覆盖：

```bash
java -jar server.jar --spring.profiles.active=prod
```

## 五、核心设计理念

### 统一响应格式

所有接口返回 `Result<T>`，前端只需要判断 `code`：

```json
{
  "code": 0,           // 0 = 成功，非0 = 失败
  "message": "登录成功",
  "data": { ... },     // 具体数据
  "timestamp": 1704067200000
}
```

### 异常处理

- 业务异常统一抛 `BusinessException`，由 `GlobalExceptionHandler` 自动转成 Result 返回
- 不需要在每个 Controller 方法里写 try-catch

### 配置分离

- 不变的配置（框架级）写在 `application-local-dev.yml` 等环境文件中
- 会变的配置（数据库地址、密码）通过变量引用，只需改文件顶部的变量

```yaml
# 变量定义在文件顶部
mysql:
  host: 127.0.0.1    # ← 改这里就行
  port: 3306
  database: operations

# 其他地方用 ${} 引用
spring:
  datasource:
    url: jdbc:mysql://${mysql.host}:${mysql.port}/${mysql.database}
```

# docker 目录脚本使用说明

> 适用场景：你站在 `server\docker\` 目录下，要发新版本、部署后端、查看进展。
> 所有脚本已统一为 **CRLF 换行符**（Windows cmd 标准），原系统就是 Windows。

---

## 1. 你之前遇到的报错根因

### 错误 A：`./apply-version.bat : 无法识别为 cmdlet...`
**PowerShell 不识别 `./`（Unix 风格）路径前缀**。这是 PowerShell 的设计。
**修法**：在 PowerShell 里用 `.\`（反斜杠）代替 `./`，或加路径前缀、或放到 PATH。

### 错误 B：bat 跑出乱码、N 行乱码、找不到命令等
**根因是 `deploy-config.bat` 含中文且 UTF-8 编码**：Windows cmd 默认 codepage 是 **936 (GBK)**，加载 .bat 时按 GBK 读字节。UTF-8 字节被当 GBK 解释 → 中文行变乱码 → `set` 静默失败 → 后续脚本读不到变量 → 雪崩式乱码。

**修法**：
- `deploy-config.bat` 改成 **全 ASCII**（配置中心不被 cmd 误解析才稳）
- 其他几个 deploy bat 含中文（用于 `echo` 显示），加 `chcp 65001 >nul` 后 cmd 在加载完后切到 UTF-8 输出
- `apply-version.bat` 也走全 ASCII（因为含 if/set/for/! 等复杂 PS 5.1 不能稳定处理中文+语法的字符），中文说明移到本 README

## 2. 三种调用方式（任选）

不管 `./` 还是 `.\`，下面的方式都能执行：

### 方式 A：双击 `apply-version.bat`（最简单）
在文件管理器里找到 `D:\work_space\website_server\server\docker\apply-version.bat`，双击。

### 方式 B：cmd 直接调用（推荐）
```cmd
C:\> cd /d D:\work_space\website_server\server\docker
C:\> apply-version.bat
```
cmd 不需要 `./` 或 `.\`，直接写 bat 文件名（不带扩展名也行，但带 `.bat` 一定行）。

### 方式 C：PowerShell 标准方式
```powershell
PS D:\work_space\website_server\server\docker> .\apply-version.bat
```
**注意：用 `.\`（反斜杠），不用 `./`（正斜杠）**——这是 PowerShell 的语法规则。

### 方式 D：PowerShell 通过 cmd 调
```powershell
PS> cmd /c D:\work_space\website_server\server\docker\apply-version.bat
```
任何写法都能用，路径可以绝对。

### 方式 E：先加到 PATH，再任意位置调用
环境变量 `PATH` 里加 `D:\work_space\website_server\server\docker`，保存后：
- cmd：`apply-version.bat` 直接执行
- PowerShell：`apply-version.bat` 直接执行

## 3. 脚本清单

| 脚本 | 编码 | 中文 | 作用 |
|---|---|---|---|
| `deploy-config.bat` | ASCII | 否（必） | **唯一配置入口**。改 `PROJECT_VERSION`、`PROJECT_ARTIFACT_ID`、服务器连接，都改这里 |

| `使用说明.md`（本文） | UTF-8 | 是 | 中文文档 |

## 4. 发新版的标准流程

### 一、把版本号改了

打开 `deploy-config.bat`，找到：
```bat
set PROJECT_ARTIFACT_ID=website_server
set PROJECT_VERSION=1.0.0
```

把 `PROJECT_VERSION` 改成你要的版本（同时也可以改 `PROJECT_ARTIFACT_ID`），保存。

### 二、同步到 pom.xml 和 Dockerfile

任选一种：
- 双击 `apply-version.bat`
- cmd：`apply-version.bat`
- PowerShell：`.\apply-version.bat`

跑完会显示 `[1/2] pom.xml synced.` `[2/2] Dockerfile synced.`，exit code 0。

### 三、构建 + 部署

```bat
cd /d D:\work_space\website_server\server
mvn clean package -DskipTests
cd docker
deploy-simple-dev.bat        :: 推到 dev
```

或生产：
```bat
deploy-simple-prod.bat       :: 推到 prod
```

部署脚本会：
1. 跑 `mvn clean package` 构建出新 jar
2. SSH 到服务器，停旧容器、删旧镜像
3. 用 `BUILD_ARTIFACT_ID` + `BUILD_VERSION` 参数 build 新镜像
4. 启新容器（挂 `/home/website_server/logs` 目录）

## 5. 仅部署（不换版本）

跳过第一步改版本、第二步同步，直接：
```bat
cd /d D:\work_space\website_server\server
mvn clean package -DskipTests
cd docker
deploy-simple-dev.bat
```

## 6. 报错速查

| 报错 | 原因 | 解决 |
|---|---|---|
| `无法将"./xxx.bat"识别为 cmdlet` | PowerShell 不识别 `./` 路径 | 用 `.\xxx.bat`、双击、cmd /c |
| `找不到 deploy-config.bat` | 当前目录不对 | `cd /d D:\work_space\website_server\server\docker` 后再跑 |
| `[X] deploy-config.bat not found!` | bat 错误信息：未配置 deploy-config.bat | 检查 deploy-config.bat 是否存在 |
| `[X] PROJECT_VERSION missing` | deploy-config.bat 没设 PROJECT_VERSION | 编辑 deploy-config.bat 设置 |
| `[X] PROJECT_ARTIFACT_ID missing` | deploy-config.bat 没设 PROJECT_ARTIFACT_ID | 编辑 deploy-config.bat 设置 |
| `[X] pom.xml not found: ..\pom.xml` | pom.xml 不在 `server\..\` 位置 | 检查 server\pom.xml 是否存在 |
| `[1/2] pom.xml synced.` 然后 [2/2] 也 synced | 正常 | – |
| 中文显示乱码 | cmd codepage 仍然 GBK | 在 cmd 里先 `chcp 65001` 再跑，或用 PowerShell |
| `SSH 连接失败` | 服务器没开 SSH 或密钥未配 | 确认 22 端口 + 密钥免密 (`ssh root@<ip>` 能直连) |

## 7. 行尾符说明

所有 `.bat` 文件现在是 **CRLF**（Windows cmd 标准）。如果用 git 拉取时被自动转成 LF，`apply-version.bat` 会无法跑。修法：
- 如果用 git：`git config core.autocrlf false`（当前项目）
- 或者 `git config --local core.autocrlf false` 只对这个项目
- 单文件修复：用编辑器（如 VS Code）打开 → 右下角显示 LF → 切换到 CRLF → 保存

---

## 8. 为什么 apply-version.bat 是英文

这个 bat 里大量 if/else/for/set/delayed-expansion `!`，**PowerShell 5.1 解析含中文+这些字符的 here-string 会崩溃**，导致 bat 内容根本写不出来。所以这个 bat 故意用 ASCII 框架，中文在这个 README 里详细说明。**功能**和**中文版**完全等价，只是 echo 用英文 `[OK]` `[X]` 而已，输出含项目变量照样能看。

## 9. 一键脚本（可选）

可以加 `build-and-deploy.bat`，一行发版：

```bat
@echo off
chcp 65001 >nul
set VERSION=%1
if "%VERSION%"=="" (
    echo Usage: build-and-deploy.bat 1.0.1
    pause
    exit /b 1
)
:: Edit deploy-config.bat's PROJECT_VERSION
call deploy-config.bat >nul 2>nul
echo Not implemented yet. Manually edit PROJECT_VERSION in deploy-config.bat, then run apply-version.bat and deploy-simple-*.bat
pause
```

但你既然用 ASCII 框架命令稳了，可以不上这层封装。

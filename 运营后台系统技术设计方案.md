## 2. 双数据源设计

### 2.1 方案
引入 `com.baomidou:dynamic-datasource-spring-boot3-starter`（与 MyBatis-Plus 同生态，最省改动）。

`application-dev.yml` 改为多数据源：
```yaml
spring:
  datasource:
    dynamic:
      primary: operations            # 默认主库（后台读写）
      strict: false
      datasource:
        operations:
          url: jdbc:mysql://122.51.175.128:23306/operations?...
          username: ydzz
          password: 'yd.zzCC#$keji'
          driver-class-name: com.mysql.cj.jdbc.Driver
        zhouyi:                       # 待用户提供，先占位
          url: jdbc:mysql://<host>:<port>/zhouyi?...
          username: <user>
          password: <pass>
          driver-class-name: com.mysql.cj.jdbc.Driver
      druid: { ... 沿用现有 Druid 池参数 ... }
```

> 若不引入 starter，则手写两个 `DataSource` + 两套 `SqlSessionFactory`/`MapperScan`（按包路径隔离 mapper）。**首选 dynamic-datasource**，更轻量。

### 2.2 使用规则
- 后台自身数据 Mapper：默认走 `operations`（primary），无需注解。
- 读 APP 数据的 Mapper/Service：方法或类上加 `@DS("zhouyi")`。
- **强约束**：`@DS("zhouyi")` 的 Mapper 一律只写 SELECT，杜绝对 zhouyi 的写操作。
- zhouyi 实体放 `com.ydzz.admin.business.entity`，与现有 `com.ydzz.entity` 隔离，避免 `type-aliases-package` 冲突。

## 3. 数据库设计（website/operations 库新增表 DDL）

> 规范：`utf8mb4`、`bigint` 主键、统一 `create_time/update_time/deleted` 逻辑删除（与现有 MyBatis-Plus 一致）。以下脚本将落地为 `server/src/main/resources/sql/admin_init.sql`。

```sql
-- 1. 后台管理员
CREATE TABLE `admin_user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(64) NOT NULL COMMENT '登录账号',
  `password` VARCHAR(100) NOT NULL COMMENT 'BCrypt 密码',
  `real_name` VARCHAR(64) DEFAULT NULL,
  `avatar` VARCHAR(255) DEFAULT NULL,
  `phone` VARCHAR(20) DEFAULT NULL,
  `email` VARCHAR(64) DEFAULT NULL,
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '0禁用 1启用',
  `last_login_time` DATETIME DEFAULT NULL,
  `last_login_ip` VARCHAR(64) DEFAULT NULL,
  `remark` VARCHAR(255) DEFAULT NULL,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted` TINYINT NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`), UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='后台管理员';

-- 2. 角色
CREATE TABLE `admin_role` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `role_code` VARCHAR(64) NOT NULL COMMENT 'SUPER_ADMIN/OPERATOR...',
  `role_name` VARCHAR(64) NOT NULL,
  `description` VARCHAR(255) DEFAULT NULL,
  `status` TINYINT NOT NULL DEFAULT 1,
  `sort_order` INT NOT NULL DEFAULT 0,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted` TINYINT NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`), UNIQUE KEY `uk_role_code` (`role_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='后台角色';

-- 3. 权限/菜单
CREATE TABLE `admin_permission` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `permission_code` VARCHAR(100) DEFAULT NULL COMMENT 'user:list 等',
  `permission_name` VARCHAR(64) NOT NULL,
  `permission_type` TINYINT NOT NULL COMMENT '1菜单 2按钮 3接口',
  `parent_id` BIGINT NOT NULL DEFAULT 0,
  `path` VARCHAR(200) DEFAULT NULL COMMENT '前端路由/接口路径',
  `component` VARCHAR(200) DEFAULT NULL,
  `icon` VARCHAR(64) DEFAULT NULL,
  `sort_order` INT NOT NULL DEFAULT 0,
  `status` TINYINT NOT NULL DEFAULT 1,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted` TINYINT NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='后台权限/菜单';

-- 4. 角色-权限
CREATE TABLE `admin_role_permission` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `role_id` BIGINT NOT NULL,
  `permission_id` BIGINT NOT NULL,
  PRIMARY KEY (`id`), KEY `idx_role` (`role_id`), KEY `idx_perm` (`permission_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='角色权限关联';

-- 5. 管理员-角色
CREATE TABLE `admin_user_role` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `admin_id` BIGINT NOT NULL,
  `role_id` BIGINT NOT NULL,
  PRIMARY KEY (`id`), KEY `idx_admin` (`admin_id`), KEY `idx_role` (`role_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='管理员角色关联';

-- 6. 登录日志
CREATE TABLE `admin_login_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `admin_id` BIGINT DEFAULT NULL,
  `username` VARCHAR(64) DEFAULT NULL,
  `login_ip` VARCHAR(64) DEFAULT NULL,
  `user_agent` VARCHAR(512) DEFAULT NULL,
  `login_type` TINYINT NOT NULL DEFAULT 1 COMMENT '1登录 2登出',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '1成功 0失败',
  `msg` VARCHAR(255) DEFAULT NULL,
  `login_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`), KEY `idx_admin` (`admin_id`), KEY `idx_time` (`login_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='后台登录日志';

-- 7. 操作日志
CREATE TABLE `admin_operation_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `admin_id` BIGINT DEFAULT NULL,
  `username` VARCHAR(64) DEFAULT NULL,
  `module` VARCHAR(64) DEFAULT NULL,
  `operation` VARCHAR(128) DEFAULT NULL,
  `method` VARCHAR(200) DEFAULT NULL,
  `request_uri` VARCHAR(255) DEFAULT NULL,
  `request_method` VARCHAR(10) DEFAULT NULL,
  `request_param` MEDIUMTEXT,
  `response_result` MEDIUMTEXT,
  `ip` VARCHAR(64) DEFAULT NULL,
  `cost_ms` BIGINT DEFAULT NULL,
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '1成功 0异常',
  `error_msg` TEXT,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`), KEY `idx_admin` (`admin_id`), KEY `idx_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='后台操作日志';

-- 8. 系统配置
CREATE TABLE `sys_config` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `config_key` VARCHAR(100) NOT NULL,
  `config_value` TEXT,
  `config_name` VARCHAR(100) DEFAULT NULL,
  `config_group` VARCHAR(64) DEFAULT NULL,
  `remark` VARCHAR(255) DEFAULT NULL,
  `update_by` VARCHAR(64) DEFAULT NULL,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`), UNIQUE KEY `uk_config_key` (`config_key`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统配置';
```

## 4. 鉴权设计（Sa-Token 独立 admin 域）

- 使用 Sa-Token 多账号体系：`StpLogic adminStpLogic = new StpLogic("admin")`，与 C 端 `StpUtil`（默认 `login`）隔离，互不挤下线。
- 封装 `AdminStpUtil`（包装 `adminStpLogic`）：`login(adminId)`、`getLoginIdAsLong()`、`logout()`。
- 拦截器 `AdminAuthInterceptor` 注册到 `/api/admin/**`（放行 `/api/admin/auth/login`）：校验 admin token。
- 权限：实现 `StpInterface`（admin 维度）返回该管理员的 `permission_code` 列表与 `role_code` 列表；接口用 `@SaCheckPermission(type="admin", value="user:list")`。
- 密码：BCrypt（hutool `BCrypt.hashpw/checkpw` 或 spring-security-crypto）。

登录流程：
```
POST /api/admin/auth/login {username,password}
  → 查 admin_user → BCrypt 校验 → 校验 status
  → AdminStpUtil.login(id) → 写 admin_login_log
  → 返回 { token, adminInfo }
GET /api/admin/auth/info  → 返回 adminInfo + roles + permissions + menuTree（前端建动态路由）
```

## 5. 操作日志（AOP）

- 注解 `@OperationLog(module="用户管理", operation="禁用用户")`。
- 切面 `OperationLogAspect` 环绕 `@OperationLog` 方法：记录 admin、module、operation、方法签名、URI、HTTP method、入参(脱敏)、出参(截断)、IP、耗时、成功/异常，异步写 `admin_operation_log`。
- 登录日志在 AuthService 内显式写入（含失败记录）。
- 脱敏：password/token 等字段在入参序列化时屏蔽。

## 6. API 设计

- 全部 `/api/admin/**`，返回 `Result<T>`。
- 分页统一 `PageResult<T> { total, records, current, size }`，查询参 `current/size + 条件`。
- 命名与方法见《运营后台系统功能分析.md 三》。
- zhouyi 依赖接口（dashboard/user/order/...）先实现 Controller+Service 骨架与 DTO，查询层标注 `// TODO 待 zhouyi 真实表`，连接到位后补 SQL。**不返回模拟数据**：未接通前返回明确的「数据源未配置」业务错误，避免假数据误导。

## 8. 前端设计

见《XiaoMaYi-EleVue资源分析.md 三》。要点：
- `request.ts` baseURL `/api`，注入 admin token，解包 `Result`，401 跳登录。
- 登录后拉 `auth/info` → Pinia 存 permissions/menu → 动态生成路由与侧边菜单。
- `v-permission` 指令做按钮级控制；后端 `@SaCheckPermission` 兜底。
- 驾驶舱用 ECharts；列表页用 ProTable + SearchForm + Pagination。
